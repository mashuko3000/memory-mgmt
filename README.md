# memory-mgmt

Репозиторий содержит комплексную программную реализацию системных компонентов. Проект выполнен в рамках учебного курса по дисциплине «Алгоритмы и структуры данных».
Система управления памятью (Allocators)

Основная часть проекта посвящена разработке кастомных аллокаторов. В качестве фундамента использована архитектура из репозитория ScarletSurge/mp_os.
Использованные компоненты:
    Интерфейсы: Базовый абстрактный класс allocator[cite: 2] и расширение allocator_with_fit_mode для поддержки различных стратегий поиска[cite: 5].
    Безопасность: Класс allocator_guardant, обеспечивающий механизм безопасного выделения и освобождения памяти через целевой аллокатор[cite: 3, 6].
    Тестирование: Утилиты allocator_test_utils для получения метаданных о состоянии блоков памяти (block_info) и проведения валидации[cite: 4, 7].

Мой вклад и реализация:
Я реализовала внутреннюю логику для следующих типов управления памятью:
    Buddy Allocator (Аллокатор парных блоков): эффективное разделение и слияние блоков для минимизации внешней фрагментации.
    Boundary Tags (Гранитные метки): реализация двусвязного списка блоков внутри выделенного пространства.
    Fit Modes: Логика выбора свободного блока согласно режимам first_fit, the_best_fit и the_worst_fit[cite: 5].

🛠 Технологии и стандарты
    C++14/17: использование современных стандартов, шаблонов (template) и механизмов std::forward для идеальной передачи аргументов в конструкторы[cite: 2].
    CMake: автоматизированная система сборки.
    Solid & Dry: следование принципам чистого кода и промышленной разработки.
